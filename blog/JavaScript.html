Same Concept, Different Name?
January 21, 2016

Ruby and JavaScript are both coding languages, so the syntax must be the same, Right?!... Unfortunately, that is way wrong!! For four weeks I learned, researched, and explored Ruby. When I finally became comfortable with Ruby, I began learning JavaScript. As you can guess from my first statement, it wasn't exactly smooth sailing. Learning Ruby and then JavaScript was in some ways a catch 22. The one advantage to this order was understanding the general concepts in Ruby and recognizing comparisons to JavaScript concepts. However, shortly after things became confusing. While reading about JavaScript, I immediately started seeing connections to Ruby. Don't be confused though. The connection can only be made between meaning and purpose. Ruby and JavaScript become two different worlds when comparing syntax.

One difference in syntax, but similarity in meaning is found when comparing methods in Ruby to functions in JavaScript. Methods and functions are generally the same thing, however they are defined differently.

JavaScript functions:

Looking at these two examples, the syntax is completely different besides the message, "hello", which each are displaying. However, both the Ruby method and JavaScript function do the same thing.

As for the Ruby example, I am defining a method called greeting by using the keyword def. Then I am telling the interpreter to put "hello" on the output screen. Then, of course I use the keyword end to close and finish my method.

Next we come to the JavaScript example. First, I am defining a function called hello() and opening that function with a left curly brace {. The parentheses that follow my function name are there to tell the computer there are no arguments or parameters. However, in other functions you may see parameters defined within these parentheses. Next, I am telling the computer to log the word "hello" to the console. Finally, I close and end my function the same way I opened it with a right curly brace }.

This is just one of the several ways Ruby and JavaScript fundamentally implement some program differently.
