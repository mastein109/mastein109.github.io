All About That Positioning!

December 20, 2015

The key to positioning in CSS is understanding the most basic concept; every single element within a page is a block. When you can envision a rectangle of pixels for each element, positioning becomes a lot easier. There are several different position attributes to learn about.

1. Static

If you do not declare a position, the element will automatically asume the default position, static. Static doesn't really tell an element to act any different than to flow into the page as it normally would. This attribute becomes useful when you are trying to remove other positioning stuck to this element.

2. Relative

Contrary to popular belief, this type of positioning just means relative to itself. This type of position can become confusing. If you do not declare other attributes to the relative element, it will be the same as leaving it static. However, if you add top, bottom, left, or right, the element will adjust accordingly to where it would normally be positioned. For examplee, if you set an element to position: relative; and left: 10px; the element will move 10px to the right of where it normally would sit.

3. Absolute

This type of positioning is very powerful and allows you to place any element exactly where you want it on a page. In addition to setting the position to absolute, you must add top, bottom, left, and right attributes to declare where you want the element placed. Although this is a very beneficial type of positioning, it comes along with cons. When you set an element to absolute, that element is removed from the natrual flow of elements within a page. It is not affected by other elements, nor does it affect any other elements. If this type of positioning is misused, it can disturb the whole entire flow of your site.

4. Fixed

This attribute to positioning is the least commonly used, but can be very helpful under certain circumstances. Fixed positioning will set an element relevant to the viewport. The viewport is a part of the webpage that stays in the same place, no matter what. For example, if a side menu is fixed on a page, it will stay fixed on the side and move with you when you scroll through. Although this is a neat concept and allows for certain elements to remain at all times, it can conflict with other elements. Depending on the size of a browser, a fixed element may overlap with other elements making it hard to read. In conclusion, if you choose fixed positioning, it is important to be cautious and test the page among several instances.
