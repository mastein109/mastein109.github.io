Week 1: The Command Line and Version Control

Git and GitHub

December 5, 2015

This week during DBC Phase 0, I learned and explored the command line and version control. Version control is very beneficial to developers for many reasons. Version control software allows you to perform baisc tasks, such as creating, saving, and editing a document, but doesn't stop there. VCS allows for several developers to work on the same project and even the same file at the same time without overwriting anyone else's work! This prevents errors and possible destruction of files, which is essential to avoid. In addition, version control software is able to track changes from all developers, so everyone can see who edited the file, where it was edited, and what was edited.
